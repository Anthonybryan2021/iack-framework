# Contributing to the I-A-C-K Framework

Thank you for your interest in contributing to the I-A-C-K Framework.

This repository is an early-stage research and prototype project focused on AI-driven cybersecurity analysis. Contributions are welcome, but they should support the project’s goals of clarity, reproducibility, responsible experimentation, and measurable improvement.

## Before You Contribute

Please review the following before opening an issue or pull request:

- Read the `README.md` to understand the project scope and current status.
- Review `SECURITY.md` before reporting any security-related concern.
- For significant changes, open an issue first so the direction can be discussed before work begins.

## Contribution Principles

Contributions should aim to:

- Improve clarity, correctness, or reproducibility.
- Keep the project aligned with its research purpose.
- Avoid introducing unsupported claims or unvalidated results.
- Preserve transparent documentation of assumptions, methods, and limitations.

## Types of Contributions

Examples of useful contributions include:

- Documentation improvements.
- Test improvements and validation coverage.
- Refactoring that improves maintainability without changing intended behavior.
- Carefully scoped feature proposals that fit the project roadmap.
- Research, metrics, and methodology improvements supported by clear reasoning.

## Development Expectations

When contributing code or documentation:

- Keep changes focused and easy to review.
- Use a feature branch instead of committing directly to `main`.
- Open a pull request with a clear summary of what changed and why.
- Include tests or validation notes when relevant.
- Update documentation when behavior, structure, or assumptions change.

## Pull Request Guidance

Before submitting a pull request, please confirm:

- The change is scoped to a clear purpose.
- Existing tests still pass, or new tests are included where appropriate.
- Documentation is updated if needed.
- The PR description explains the motivation, implementation, and any tradeoffs.

## Security

Please do **not** open public issues for vulnerabilities or sensitive security concerns.

Instead, follow the reporting guidance in `SECURITY.md`.

## Conduct

Contributors should communicate respectfully, stay constructive, and keep discussions grounded in the project’s mission and evidence-based improvement.

## Maintainer Review

All contributions are reviewed for alignment with the project’s goals, technical quality, and research integrity. Submission does not guarantee merge, but thoughtful contributions are appreciated.
