# Security Policy

## Supported Versions

The I-A-C-K Framework is an early-stage research and prototype project. Security support applies to the current `main` branch and the latest published revision.

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue. Instead, report it privately to the project maintainer.

Please include:
- a clear description of the issue,
- the affected file, feature, or workflow,
- steps to reproduce, if available,
- and any suggested fix or relevant evidence.

Reports will be reviewed as soon as possible, and responsible disclosure is appreciated.

## Scope

This policy applies to the repository code, documentation, and GitHub workflows associated with the I-A-C-K Framework.
